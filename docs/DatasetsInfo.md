Состоит датасет из тренировочных и проверочных данных.

Сами данные представляют из себя набор изображений и json файла. 

Информация об изображениях:
Изображения были получены в результате съемки лидара, который был установлен на автомобиле.
Список объектов, которые могут быть изображены на изображениях:
1. Грузовик
2. Автомобиль
3. Поезд
4. Проезжая часть
5. Полоса движения
6. Дорожный знак
7. Автобус
8. Светофор
9. Велосипед
10. Человек

Информация о json:
Json файл представляет из себя огромный массив объектов со следующими свойствами:
1. name - имя картинки
2. attributes - список признаков картинки
	2.1. weather - погода
	2.2. scene - место действия (улица, шоссе и тд)
	2.3. timeofday - часть суток (утро, день, вечер, ночь)
3. timestamp - отметка времени (судя по всему, отметка времени видео, из которого получено изображение)
4. labels - метки на изображении для фиксации объекта.
	Базовые данные для метки:
	4.1. category - категория объекта 
	4.2. attributes - атрибуты, по сути свойства объекта, которые подтверждают принадлежность объекта к определенному классу. У разного класса может быть разный перечень этих атрибутов. Например, у светофора может быть атрибут текущего горящего цвета.
	4.3. manualShape - не до конца понятно для чего это свойство нужно. Возможно, для обозначения, что форма для обводки объекта нестандартное.
	4.4. manualAttributes - аналогично. Возможно, что среди атрибутов присутствуют нестандартные.
	4.5. Геометрия:
		4.5.1. box2d - объект, который хранит координаты диагонали прямоугольника для обводки объекта. Используется для подавляющего большинства объектов, кроме обозначения дороги и прочего.
		4.5.2. poly2d - объект, описывающий полигон. Имеет свойство vertices для хранения информации о координатах вершин полигона для обводки объекта (в основном для всяких дорог используется)
	4.6. id - идентификатор метки. Нигде не нашлось повторяющейся метки.
